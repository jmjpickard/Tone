# Skill: briefing

## Purpose
Prepare a concise morning briefing with priorities and context.

## Trigger Patterns
- "brief me"
- scheduled morning run
- "what matters today"

## Input Format
- date: local day
- sources: tasks, recent threads, daily notes

## Output Format
- priorities
- active threads
- pending tasks

## Examples
- Input: scheduled run at configured briefing time
- Output: three-section summary with recommended first actions

## Constraints
- Prioritize relevance over completeness.
- Keep structure stable for skim-readability.
- Note uncertainty instead of guessing.
