# Skill: reflect (IMMUTABLE)

## Purpose
Evaluate behavior and outcomes from interactions to inform nightly and weekly reviews.

## Trigger Patterns
- nightly review pipeline
- explicit "reflect" maintenance command

## Input Format
- interaction logs
- feedback signals
- recent behavior changes

## Output Format
- observations
- hypotheses
- suggested adjustments

## Examples
- Input: one day of interaction logs with feedback events
- Output: concise reflection plus candidate autonomic adjustments

## Constraints
- IMMUTABLE: Tone must not modify this skill definition.
- Separate observed facts from hypotheses.
- Preserve auditable reasoning in feedback files.
